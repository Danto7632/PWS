저는 신규 입사자·아르바이트생이 짧은 시간에 실전처럼 배우도록, 종이/문서 메뉴얼을 바로 “대화형 시뮬레이션”으로 바꾸는 도구를 Python·Streamlit으로 만듭니다. 기획–설계–개발–운영을 한 흐름으로 묶고, 즉시 피드백·재연습 루프를 중시합니다.

본 프로젝트는 고객/직원 역할을 양방향으로 바꿔가며 연습하는 AI 시뮬레이터입니다. 매장/직무 메뉴얼을 업로드하면, 고객 질문을 생성하고(변형·응용 포함), 직원 답변의 정확성·친절도·정책 준수 여부를 자동 피드백합니다. 복잡한 인프라 대신 Streamlit 멀티페이지 구조로 빠르게 배포·운영할 수 있게 설계했습니다.

실전형 업무 시뮬레이터 for 신입

(고객응대 트레이닝)

[문제 정의]

신입은 전화/현장 응대 경험이 부족해 긴장·실수·응답 지연이 잦고, 메뉴얼은 “읽고 끝”인 경우가 많아 정책 준수/표현 톤이 일관되지 않습니다. 교육자는 반복 코칭에 시간이 부족하고 케이스 다양화도 어렵습니다. 이 프로젝트는 AI가 고객·직원 역할을 모두 수행하고, 메뉴얼 기반+변형 질문으로 연습 난이도를 조절하며, 자동 피드백으로 적응 시간을 단축합니다.

[전체 기술 스택]

프론트엔드:
Streamlit(Python UI), Markdown 렌더링, 필요 최소 HTML/CSS 커스터마이징, Plotly(로그/지표 시각화)

백엔드:
Python 3.11, Streamlit Session State, SQLite(기본) / PostgreSQL(선택), SQLAlchemy, Pydantic

DevOps & Tools:
Git/GitHub, Streamlit Community Cloud(또는 EC2 Docker), OpenAI/Exaone 연동 어댑터, 간단 RAG(FAISS/Chroma 선택), pytest, Black/Ruff

[아키텍처/설계]

단일 코드베이스의 Streamlit 멀티페이지 앱 구조를 채택하여 효율적인 개발과 배포를 목표로 합니다.

  • 자료 업로드/지식화: 매장/직무 메뉴얼 업로드 → 간단 임베딩 인덱스(FAISS/Chroma 선택) → RAG 컨텍스트 구성
  • 시뮬레이션: (모드 A) AI=고객, 사용자가 직원으로 응답 / (모드 B) AI=직원, 사용자가 고객 질문
  • 피드백: 규칙 체크(금지어/정책 누락/금액·절차 오류) + LLM 루브릭(정확성·친절도·컴플라이언스) → 항목별 코멘트/재연습 가이드
  • 데이터 저장: SQLite(기본) 또는 PostgreSQL(선택)로 시나리오·대화로그·점수·재시도 이력 관리
  • 배포/운영: Streamlit Cloud(간편) 또는 Docker+EC2(사내 배포). 모델 어댑터는 교체형 인터페이스(OpenAI/Exaone).

[핵심 기능]

고객 역할 AI 시뮬레이션

챗봇 형식으로 다양한 고객 페르소나·상황을 생성(예: 초기 불만/환불/재고 문의/예약 변경). 메뉴얼 근거로 정답 범위를 유지하면서, 표현만 변형된 응용 질문을 섞어 암기식 대응을 방지합니다. 난이도(톤/압박감/정보 누락 정도)와 주제 태그로 세션을 구성합니다.

직원 역할 AI 응대 기능

사용자가 고객이 되어 질문하면, AI가 메뉴얼 준수형 직원 응답을 생성합니다. 절차/정책/요금을 자동 참조하고, 사과/공감 템플릿을 톤 가이드에 맞춰 반영합니다. 사용자는 고객 관점에서 바람직한 응대 흐름을 체험하고, 좋은 답변의 구조를 학습합니다.

역할 전환·재연습 루프

버튼 한 번으로 고객↔직원 전환. 세션 종료 시 항목별 점수+코칭 포인트 제공, 부족 영역만 퀵 재연습(동일 의도·다른 문장) 지원. 누적 지표(정확성/톤/정책 준수)와 최근 3회 개선율을 카드로 표시하여 학습 효과를 극대화합니다.

[나의 역할 및 기여]

  • Streamlit 멀티페이지 설계(업로드→시뮬레이션→피드백→통계), 상태관리(Session State) 구현
  • 메뉴얼 RAG 파이프라인(간단 임베딩 인덱스, 섹션/정책 태깅) 및 루브릭 스키마 정의
  • 금지어/누락 항목 등 규칙 검증 + LLM 루브릭 평가 병합 로직, 피드백 뷰(항목별 코멘트/개선 TODO)
  • 배포 스크립트(Streamlit Cloud/EC2 Docker) 및 데이터 마이그레이션 가이드 정리

[결과 및 성과]

  • 파일럿 목표: 초기 응답 시간 30% 단축, 정책 누락률 40%↓, 재연습 3회 내 합격률 80%+ (초기 지표 수집 예정)
  • 교육자 관점: 개별 코칭 시간 절감, 케이스 다양화 자동화, 교육 로그로 현황 파악 용이
  • 학습자 관점: 긴장 완화, 자신감 향상, 표현 톤 표준화 및 에스컬레이션 기준 명확화

[회고]

배운 점

메뉴얼의 구조화(정책/예외/톤 가이드 분리)가 모델 품질보다 중요합니다. 질문을 “의도·정책·감정 톤”으로 태깅하니 난이도 조절과 피드백 명확성이 크게 개선되었습니다. 또한 Streamlit만으로도 운영 가능한 속도로 MVP→수업 투입까지 연결할 수 있었습니다.

개선점

음성(STT/TTS) 모드, 상황 카드 에디터(비매너/컴플레인 시나리오 조합), 현장 KPI 연동(CS 만족/콜백률)과 개인화 난이도 곡선(최근 오류 태그 기반)을 추가해 실전 적합도를 더 높일 계획입니다.
